e YouTube Trend Intelligence MVP (v1)

## 1. 이 서비스는 무엇인가

이 서비스는 **유튜브에서 지금 ‘뜨고 있는 영상(특히 Shorts)’을 빠르게 탐지하고,
왜 뜨는지 근거를 제공하여 크리에이터의 콘텐츠 기획을 돕는 웹 서비스**이다.

- 단순 통계 나열이 아닌,
- “지금 어떤 영상이 뜨고 있는가”와
- “이 영상이 급등하는 이유는 무엇인가”를
**데이터 기반으로 보여주는 것**이 목적이다.

---

## 2. MVP 1차의 명확한 범위

### ✅ 포함 (Must)
- 플랫폼: **YouTube only**
- 지역/언어: **대한민국**
- 대상 콘텐츠: **YouTube 영상 (Shorts 중심, 일반 영상 포함 가능)**
- 핵심 가치: **급등 영상 탐지 + 근거 제공**
- 접근 방식: **전체 트렌드 기반 + 내 채널 분석 둘 다 제공**

### ❌ 제외 (Not Now)
- Instagram / Threads / TikTok
- 글로벌 데이터
- 유료 모델
- 복잡한 추천 알고리즘
- 다중 분석 윈도우(24H, 7D 등)

---

## 3. 핵심 사용자

### Primary User
- **유튜브 크리에이터**
  - 매일 또는 주기적으로 콘텐츠를 제작
  - “지금 뜨는 소재”를 빠르게 찾고 싶어함
  - 감(직감)이 아닌 **근거 있는 트렌드**를 원함

---

## 4. MVP의 핵심 문제 정의

> ❓ 지금 유튜브에서 어떤 영상이 뜨고 있는지  
> ❓ 그 영상은 왜 갑자기 뜨고 있는지  
> ❓ 내가 참고하거나 비교할 만한 영상은 무엇인지  

→ 이 3가지를 **한 화면 흐름 안에서 해결**하는 것이 MVP의 목적이다.

---

## 5. MVP 핵심 기능 (필수 기능만)

### 5.1 급등 영상 탐지 (Core)
- 유튜브 공식 **카테고리 ID 기준**
- 최근 **8시간 기준 급등 영상 리스트 제공**
- 급등 기준:
  - 조회수 / 좋아요 증가 속도가
  - 동일 카테고리·유사 업로드 시점 영상 대비
  - 상대적으로 빠른 경우

> 결과는 “급등”이라는 **라벨 + 설명 문장**으로 제공

---

### 5.2 영상 상세 분석
각 영상에 대해 다음을 제공한다:

- 기본 메타 정보
  - 영상 ID
  - 제목
  - 설명
  - 태그
  - 업로드 날짜
  - 채널 정보
- 시계열 데이터
  - 조회수 변화
  - 좋아요 변화
- 급등 판단 설명
  - 왜 급등으로 판단되었는지에 대한 문장 설명

---

### 5.3 전체 트렌드 기반 탐색
- 카테고리별 급등 영상 리스트
- 정렬 기준:
  - 조회수 증가 속도
  - 좋아요 증가 속도
- “지금 참고할 만한 영상”을 빠르게 찾는 용도

---

### 5.4 내 채널 분석 (MVP 버전)
- 사용자가 자신의 유튜브 채널을 입력
- 최근 업로드 영상의:
  - 조회수/좋아요 추이 확인
  - 전체 트렌드 영상과의 비교 가능

> 정교한 성과 분석이 아니라  
> **“트렌드 대비 내 위치”를 감각적으로 보여주는 수준**

---

### 5.5 영상 비교
- 영상 A vs 영상 B
- 비교 항목:
  - 조회수 증가 추이
  - 좋아요 증가 추이
  - 업로드 시점
  - 길이 / 제목 / 태그

---

## 6. 데이터 수집 원칙 (MVP 기준)

### 수집 대상
- 유튜브 인기 영상 / 검색 결과 기반으로 발견된 영상
- 급등 후보 영상 위주

### 수집 데이터
- 영상 메타 정보 (고정)
- 조회수 / 좋아요 수 (변동)
- 시계열 스냅샷

### 수집 주기
- **기본 10분 단위**
- 단, **모든 영상이 아닌 ‘선별된 영상’만 추적**
  - 급등 후보
  - 상위 트렌드 영상
  - 사용자가 조회/비교한 영상

> ❗ 비용/쿼터 이슈를 고려하여  
> “전수 수집”은 하지 않는다.

---

## 7. KPI (MVP 성공 기준)

### 핵심 KPI
- **사용자가 서비스에서 확인한 영상의 조회수(View Count)**

의미:
- 사용자가 실제로 “볼 가치가 있는 영상”을 발견했는지
- 트렌드 탐지의 실효성이 있었는지를 간접적으로 판단

---

## 8. 기술 스택 (확정)

- Frontend: **Next.js**
- Backend: **Python (FastAPI)**
- Database:
  - Main: **Supabase (PostgreSQL)**
  - NoSQL: MongoDB (MVP에서는 사용하지 않음)
- 플랫폼 로그인:
  - 카카오 로그인 (우선)

---

## 9. MVP의 성공 기준 (한 문장)

> “유튜브 크리에이터가 하루에 한 번이라도 들어와  
> ‘아, 이 영상은 참고할 만하다’라고 느끼게 만드는 서비스”

---

## 10. 이 문서의 역할

- 이 문서는 **MVP 1차 개발의 기준선**이다.
- 기능 추가/확장은 이 문서를 **침범하지 않는 선에서** 진행한다.
- 개발/기획/디자인 논의 시  
  → “이게 MVP 핵심인가?”를 판단하는 기준으로 사용한다.
